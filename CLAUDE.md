# NotifyHub — Working Notes

Before starting any task, read `CODEBASE_MAP.md` first instead of re-scanning the codebase.
Only search actual files when the map doesn't answer the question or looks stale for the
area being touched.

After any change that adds/changes entities, endpoints, jobs, key business logic, or tests,
update the relevant section of `CODEBASE_MAP.md` as part of that same change — not a
separate follow-up task, and not a full rewrite, just an update to what changed.

If `CODEBASE_MAP.md` ever contradicts the actual code, trust the code, fix the map, and
flag the discrepancy.
